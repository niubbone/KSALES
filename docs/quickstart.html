1. Import mensile completo

Sequenza standard da eseguire ogni mese dopo aver ricevuto i file da WKI.

Passo A — Carica i file in Google Drive

  1. Caricare i CSV rendiconti (dml*.csv) nella cartella Drive rendiconti
  2. Caricare i dips148*.csv nella cartella Drive S148
  3. Caricare i ripf502*.xlsx nella cartella Drive F502
  4. Caricare i file BudAnal (*.xlsx estrazioni) nella cartella Drive estrazioni (se disponibili)

Passo B — Esegui le 3 fasi (Apps Script → Esegui)

Fase 1
flusso_1_import
Importa S148 + F502 + rendiconti DML + estrazioni BudAnal dalla cartella Drive.
Fase 2
flusso_2_popola
Aggiunge nuovi clienti e contratti, backfill date firma e num_contratto, aggiorna stati.
Fase 3
flusso_3_genera
Rigenera tutti i tab ORD e PAG nel foglio CS, svuota la cache GAS.
Ogni fase è idempotente: se va in timeout, rilanciarla — ricomincia senza duplicare i dati. Vedi §2 per la procedura di ripristino.

Passo C — Aggiorna il frontend

Frontend → "Aggiorna tutto"   → svuota localStorage e ricarica i dati da GAS

2. Gestione timeout

Apps Script ha un limite di 6 minuti per esecuzione. Se una fase va in timeout:

  1. Notare quale fase si è bloccata (FASE1 / FASE2 / FASE3 nel Logger)
  2. Rilanciarla direttamente — ogni operazione è idempotente, non crea duplicati
  3. Se FASE1 importa molti file, usare _importaAnnoSlice per importare i rendiconti a blocchi
Le fasi 2 e 3 sono solitamente veloci (<2 min). Se va in timeout la Fase 1 sull'import rendiconti, è normale — rilanciare o usare lo slice.

3. Solo rendiconti (aggiornamento rapido)

Quando hai solo nuovi rendiconti da importare e non ci sono nuovi contratti o estrazioni.

Puoi eseguire tutte e 3 le fasi normalmente (sono idempotenti), oppure usare le funzioni singole:

_importS148Folder          → importa nuovi dips148*.csv
importaRendiconti2026      → importa nuovi dml*.csv
_populateContratti         → aggiunge contratti eventualmente nuovi
_backfillNumContratto      → ricalcola aggregazione (idempotente)
_aggiornaStatiContratti    → aggiorna stati
_generaCS                  → rigenera CS
clearAllCache              → svuota cache GAS

Frontend → "Aggiorna tutto"

4. Solo estrazioni (aggiornamento sub-agenti)

Quando hai aggiornamenti da BudAnal ma non nuovi rendiconti.

importEstrazioniFolder
_backfillSubAgenti
_sincronizzaAgenzieDaClienti
_generaCS
clearAllCache

Frontend → "Aggiorna tutto"

5. Riconciliazione e segnalazioni

Importa segnalazioni da CS

1. _generaCS                    → assicurati che il CS sia aggiornato
2. _importaProblemiDaCS         → importa contratti con saldo aperto in SEGNALAZIONI
3. _formatSegnalazioni          → applica formattazione e colori (solo se i colori mancano)

Genera lettera di contestazione WKI

_generaLetteraSegnalazioni            → tutte le segnalazioni APERTA/LAVORAZIONE
_generaLetteraSegnalazioniMese        → solo un mese specifico (editare MESE nel codice)

Chiudi segnalazione risolta

Dalla tab SEGNALAZIONI: selezionare CHIUSA nella colonna A. La riga si sposta automaticamente in SEGNALAZIONI_CHIUSE.

In alternativa via script (editare l'ID nel codice):

_chiudiSegnalazione   → es. 'SEG-0012'

Report annuale

_reconciliazioneAnnuale   → riepilogo per linea: provvigioni / fatture / pagamenti (editare ANNO)

8. Setup iniziale (una tantum)

Solo per nuove installazioni o ripristino del DB.

  1. Eseguire setupDB → crea tutte le tab nel KS-DB
  2. Eseguire populateProdotti → popola il catalogo prodotti con le aliquote
  3. Aprire Codice.js → funzione _impostaCS → sostituire 'ID_FOGLIO_CS' con l'ID reale del Google Sheet CS → eseguire
  4. Verificare gli ID cartelle Drive in flusso.gs (FOLDER_REND, FOLDER_F502, FOLDER_ESTR)
  5. Eseguire l'import mensile completo (punto 1)
Il foglio CS è separato dal KS-DB. L'ID si trova nell'URL: docs.google.com/spreadsheets/d/ID_QUI/edit
