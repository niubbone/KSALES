1. Import mensile completo

Sequenza da eseguire ogni mese dopo aver ricevuto i file da WKI.

Fase 1 — Carica i file

  1. Caricare i CSV rendiconti (dml*.csv) nella cartella Drive rendiconti
  2. Caricare i dips148*.csv nella cartella Drive S148
  3. Caricare i ripf502*.xlsx nella cartella Drive F502
  4. Caricare i file BudAnal (*.xlsx estrazioni) nella cartella Drive estrazioni (se disponibili)

Fase 2 — Importa in KS-DB (Apps Script → Esegui)

1. _importS148Folder        → importa dips148*.csv (linea S148)
2. _importF502Folder        → importa ripf502*.xlsx (linea F502)
3. importRendicontiFolder   → importa CSV rendiconti (linee G174/X136/X200)
4. importEstrazioniFolder   → importa BudAnal (facoltativo, necessario per sub-agenti)
Importare SEMPRE prima S148/F502/rendiconti e DOPO gli altri passaggi. I nuovi contratti devono essere in RENDICONTI prima di essere aggiunti a CONTRATTI.

Fase 3 — Aggiorna le tabelle master

5. _populateClienti              → aggiunge nuovi clienti a CLIENTI
6. _populateContratti            → aggiunge nuovi contratti a CONTRATTI
7. _backfillDataFirmaContratti   → popola data_firma da data_doc SAP (non sovrascrive date esistenti)
8. _backfillNumContratto         → raggruppa ordini G+NT nello stesso contratto commerciale
9. _backfillSubAgenti            → aggiorna codice_agenzia da BudAnal (solo se sono stati importati)
10. _sincronizzaAgenzieDaClienti → propaga codice_agenzia da CLIENTI → CONTRATTI

Fase 4 — Genera report e stati

11. _generaCS                    → rigenera tab ORD e PAG nel foglio CS
12. _aggiornaStatiContratti      → aggiorna stati contratti (SALDATO/OVER/ANOMALIA/ATTIVO)

Fase 5 — Aggiorna frontend

13. clearAllCache                → svuota cache GAS (det_... + comm_...)
14. Frontend → "Aggiorna tutto"  → svuota localStorage e ricarica da GAS

2. Solo rendiconti (aggiornamento provvigioni)

Quando hai solo nuovi rendiconti da importare, senza nuovi contratti.

1. importRendicontiFolder      → importa i nuovi CSV
2. _generaCS                   → rigenera CS
3. _aggiornaStatiContratti     → aggiorna stati
4. clearAllCache + "Aggiorna tutto"

3. Solo estrazioni (aggiornamento sub-agenti o contratti)

Quando hai aggiornamenti da BudAnal ma non nuovi rendiconti.

1. importEstrazioniFolder
2. _backfillSubAgenti
3. _sincronizzaAgenzieDaClienti
4. _generaCS
5. clearAllCache + "Aggiorna tutto"

4. Riconciliazione e segnalazioni

Importa segnalazioni da CS

1. _generaCS                    → assicurati che il CS sia aggiornato
2. _importaProblemiDaCS         → importa contratti con saldo aperto in SEGNALAZIONI
3. _formatSegnalazioni          → applica formattazione e colori (solo se i colori mancano)

Genera lettera di contestazione WKI

_generaLetteraSegnalazioni            → tutte le segnalazioni APERTA/LAVORAZIONE
_generaLetteraSegnalazioniMese        → solo un mese specifico (editare MESE nel codice)

Chiudi segnalazione risolta

_chiudiSegnalazione   → editare l'ID segnalazione nel codice (es. 'SEG-0012')

Report annuale

_reconciliazioneAnnuale   → riepilogo per linea: provvigioni / fatture / pagamenti (editare ANNO)

6. Checklist anomalie frequenti

SintomoCausa probabileFix
Frontend mostra dati vecchi Cache localStorage o GAS non svuotata «Aggiorna tutto» nel frontend
Nuovi contratti non compaiono _populateContratti non eseguito dopo l'import Eseguire _populateContratti poi svuotare cache

Data firma mancante o errata data_firma non propagata da RENDICONTI Eseguire _backfillDataFirmaContratti

Molti ANOMALIA su contratti recenti data_firma mancante (= data 0 o invalida) Eseguire _backfillDataFirmaContratti poi _aggiornaStatiContratti
Badge SYN_ (senza n_doc) nel frontend Contratto recente — n_doc SAP non ancora nei rendiconti Atteso; scomparirà al prossimo import quando SAP fornisce n_doc reale
Segnalazione già risolta compare ancora Segnalazione non chiusa o CS non rigenerato _chiudiSegnalazione('SEG-XXXX') oppure _importaProblemiDaCS dopo _generaCS
Errore "Tab RENDICONTI mancante" setupDB non eseguito Eseguire setupDB una volta sola

7. Setup iniziale (una tantum)

Solo per nuove installazioni o ripristino del DB.

  1. Eseguire setupDB → crea tutte le tab nel KS-DB
  2. Eseguire populateProdotti → popola il catalogo prodotti con le aliquote
  3. Aprire Codice.js → funzione _impostaCS → sostituire 'ID_FOGLIO_CS' con l'ID reale del Google Sheet CS → eseguire
  4. Eseguire l'import mensile completo (punto 1)
Il foglio CS è separato dal KS-DB. L'ID si trova nell'URL: docs.google.com/spreadsheets/d/ID_QUI/edit
