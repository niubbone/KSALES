1. Flusso mensile standard

Ogni mese, dopo aver ricevuto i file da WKI:

PassoFonteAzione
1. Import rendiconti PDF+CSV Python Scarica PDF e CSV in cartella locale → python import_rendiconti.py
2. Import CSV rendiconti aggiuntivi (opz.) Frontend Import → Carica CSV rendiconto (dml*.csv, dips148*.csv)
3. Import estrazione BudAnal Frontend Import → Carica file BudAnal → seleziona .xls SAP
4. Avvia Riconciliazione Frontend Import → Avvia Riconciliazione (ricalcola FIRMA/INCASSI/ESTRAZIONI)
5. Popola e genera CS (se servono nuovi contratti) GAS flusso_2_popola → flusso_3_genera
I passi 2–4 sono ora eseguibili direttamente dal frontend senza aprire Apps Script. Il passo Python (1) è necessario per l'import dei PDF (mappatura NT→GF da SEZ.3). Per i soli CSV, si può usare il frontend (passo 2) senza aprire il terminale.

2. Import rendiconti

Percorso A — Pipeline Python (PDF + CSV, raccomandato per import mensile completo)

  1. Scarica i file WKI ricevuti via email nella cartella locale rendiconti:
    rendiconti/YYYY/MM/LINEA/dml*.pdf e dml*.csv
    rendiconti/YYYY/MM/S148/dips148*.csv
  2. Apri un terminale nella cartella pipeline ed esegui: 
    python import_rendiconti.py
    Flag: --dry-run (simula senza inviare), --reset (riprocessa tutto), --pdf-only (solo PDF/SEZ.3), --csv-only (solo CSV)
La pipeline è idempotente: ogni file viene tracciato in state/processed.json e non viene inviato due volte. Per forzare il re-import di un file specifico, eliminalo dal JSON o usa --reset.

Percorso B — Upload dal frontend (CSV singoli, senza aprire il terminale)

Utile quando hai pochi file da caricare o vuoi evitare il terminale per i soli CSV.

  1. Frontend → sezione Import
  2. Click su Carica CSV rendiconto → seleziona uno o più file dml*.csv o dips148*.csv
  3. Il frontend rileva automaticamente il tipo dal nome file (S148 vs standard) e invia a GAS
Il percorso B non processa i PDF: la mappatura NT→GF (SEZ.3) viene saltata. Usarlo solo se non ci sono nuovi contratti triennali o se il percorso A è già stato eseguito per i PDF.

Import F502 (xlsx mensili)

F502 usa file Excel con due sheet (Produzione + Incassi). Import manuale da GAS:

_importF502Folder()   → importa tutti i ripf502*.xlsx dalla cartella Drive F502

3. Import estrazione BudAnal

L'estrazione SAP (portafoglio contratti attivi) arriva come file .xls con encoding HTML/MIME SAP.

  1. Frontend → sezione Import
  2. Click su Carica file BudAnal → seleziona il file *.xls
  3. Il file viene inviato in base64 a GAS → parsato → inserito in ESTRAZIONI (con dedup)
  4. Se configurato ESTRAZIONI_FOLDER_ID, il file viene archiviato anche in Drive con timestamp
La dedup key è: data_estrazione | linea | cod_cliente | desc_materiale. Ricaricare lo stesso file non crea duplicati.

4. Avvia Riconciliazione

Ricalcola i tre tab di riconciliazione (FIRMA, INCASSI, ESTRAZIONI) nel foglio KS-DB e svuota la cache frontend.

  1. Frontend → sezione Import
  2. Click su Avvia Riconciliazione
  3. Attendi il completamento (può richiedere 30–90 secondi su dataset grandi)
Logica firma (luglio 2026): il check è sul totale cumulato firma ricevuta vs firma attesa (imponibile × aliq da PRODOTTI). Non esiste più la categoria FIRMA_DILAZIONATA: se il totale torna algebricamente è OK, indipendentemente da quante tranche sono arrivate.
Stati riconciliazione firma: OK | ANOMALIA | FIRMA_MANCANTE | SENZA_ALIQ | VECCHIO_MANDATO

5. Riconciliazione CS e segnalazioni (GAS)

Rigenera foglio CS e importa anomalie

1. flusso_2_popola          → aggiunge nuovi clienti/contratti, backfill, aggiorna stati
2. flusso_3_genera          → rigenera tab ORD e PAG nel foglio CS
3. _importaProblemiDaCS     → importa contratti con saldo aperto in SEGNALAZIONI
4. _formatSegnalazioni      → applica formattazione e colori (solo se mancano)

Genera lettera di contestazione WKI

_generaLetteraSegnalazioni            → tutte le segnalazioni APERTA/LAVORAZIONE
_generaLetteraSegnalazioniMese        → solo un mese (editare MESE nel codice)

Chiudi segnalazione risolta

Dalla tab SEGNALAZIONI: selezionare CHIUSA nella colonna A → la riga si sposta automaticamente in SEGNALAZIONI_CHIUSE.

8. Setup iniziale (una tantum)

Solo per nuove installazioni o ripristino del DB.

  1. Eseguire setupDB → crea tutte le tab nel KS-DB
  2. Eseguire populateProdotti → popola il catalogo prodotti con le aliquote
  3. Aprire Codice.js → _impostaCS → sostituire 'ID_FOGLIO_CS' con l'ID reale del Google Sheet CS → eseguire
  4. Verificare gli ID cartelle Drive in flusso.gs (FOLDER_REND, FOLDER_F502, FOLDER_ESTR)
  5. Impostare proprietà script ESTRAZIONI_FOLDER_ID per abilitare l'archiviazione Drive degli upload BudAnal (opzionale)
  6. Eseguire il flusso mensile completo (punto 1)
Il foglio CS è separato dal KS-DB. L'ID si trova nell'URL: docs.google.com/spreadsheets/d/ID_QUI/edit
